Flux Widgets

A declarative, Flutter-inspired widget toolkit for building native Windows UIs. Chain methods, compose layouts, and bind reactive state.

Text

Display

Renders a string of text. Auto-sizes to its content by default.

// Basic usage
auto label = Text("Hello, world!")
    ->setFontSize(18)
    ->setFontWeight(FontWeight::Bold)
    ->setTextColor(RGB(30, 30, 30))
    ->setPadding(12);

// Bound to state
auto counter = Text(myState);

Factory

SignatureDescription
Text(string)Static text
Text(State<T>)Reactive text, auto-updates when state changes
Text(State<T>, trueStr, falseStr)Toggle between two strings based on bool state

Button

Interaction

A clickable widget with a background. Accepts either a text label or a child widget for fully custom content.

// Text button
auto btn = Button("Save", [&](){
    // handle click
})
    ->setBackgroundColor(RGB(59, 130, 246))
    ->setHoverBackgroundColor(RGB(37, 99, 235))
    ->setBorderRadius(6)
    ->setPadding(12);

// Widget child button
auto iconBtn = Button(Text("★ Star")
    ->setFontSize(16), onClick);

Factory

SignatureDescription
Button(string, onClick?)Text-only button with optional click handler
Button(WidgetPtr, onClick?)Custom child widget inside the button

Row

Layout

Lays out children horizontally. Supports flex expansion and alignment on both axes.

auto row = Row(
    Text("Left"),
    Expanded(SizedBox(0, 0)),  // spacer
    Text("Right")
)
    ->setSpacing(12)
    ->setCrossAlignment(Alignment::Center);

Methods

setBackgroundColor(color)COLORREFBackground fill
setFlex(f)intFlex factor when inside Column

Column

Layout

Lays out children vertically. Works symmetrically to Row — identical API, vertical axis.

auto col = Column(
    Text("Title")->setFontSize(20),
    Text("Subtitle"),
    Expanded(contentWidget)
)
    ->setSpacing(8)
    ->setCrossAlignment(Alignment::Stretch);

Container

Layout

A single-child box widget with full styling support — borders, background, padding, margin, constraints, and event handlers. The most versatile primitive.

auto box = Container(child)
    ->setBackgroundColor(RGB(255, 255, 255))
    ->setBorderColor(RGB(200, 200, 200))
    ->setBorderWidth(1)
    ->setBorderRadius(8)
    ->setPadding(16)
    ->setMinWidth(200)
    ->setOnClick([&](){ /* ... */ });

Center

Layout

Centers a single child both horizontally and vertically within the available space.

auto centered = Center(
    Text("I'm centered!")
);
Note: Center expands to fill all available space by default, then centers its child within that space.

Expanded

Layout

Makes a child fill the remaining space inside a Row or Column. Use the flex parameter to control proportional sizing between multiple Expanded widgets.

Row(
    Expanded(panelA, 2),  // takes 2/3 of space
    Expanded(panelB, 1)   // takes 1/3 of space
);

Factory

SignatureDescription
Expanded(child, flex = 1)Wrap a child to make it flex-expand

SizedBox

Layout

A fixed-size box. Use it to constrain a child widget's dimensions, or as an empty spacer.

// Spacer between items
SizedBox(16, 0);

// Constrained child
SizedBox(200, 48, myWidget);

Factory

SignatureDescription
SizedBox(width, height, child?)Fixed-dimension box, child is optional

Padding

Layout

A convenience shorthand for wrapping a widget in uniform padding. Equivalent to a Container with only padding set.

Padding(24, myWidget);

Factory

SignatureDescription
Padding(amount, child)Wraps child with uniform padding on all sides

Divider

Display

A 1px horizontal rule that fills the available width. Used to visually separate sections.

Column(
    sectionA,
    Divider(),
    sectionB
);

Scaffold

Structure

The root-level page wrapper. Composes an optional AppBar with a body that fills the remaining space.

auto page = Scaffold(
    AppBar("My App"),
    Center(Text("Hello!"))
);

Factory

SignatureDescription
Scaffold(appBar?, body?)Both parameters are optional

AppBar

Structure

A 56px top navigation bar with a built-in title. Blue background with white bold text out of the box.

AppBar("Settings");

Factory

SignatureDescription
AppBar(title)Creates an AppBar with the given title string

Card

Structure

A pre-styled elevated container: white background, light border, 8px rounded corners, and 16px padding on all sides.

Card(
    Column(
        Text("Card Title")->setFontSize(18),
        Text("Some content here.")
    )->setSpacing(8)
);

Factory

SignatureDescription
Card(child)Wraps child in a styled card container

State Binding

Reactive

Many widget properties accept a State<T> object. When state changes, bound widgets automatically re-render — no manual refresh needed.

// Declare state
State<bool> isActive{false};
State<int> count{0};

// Bind text to state
Text(count);  // shows "0", updates automatically

// Toggle text based on bool state
Text(isActive, "ON", "OFF");

// Bind background color
Container(child)
    ->setBackgroundColor(isActive,
        RGB(74, 222, 128),   // true: green
        RGB(248, 113, 113)    // false: red
    );

// Update state on click → widgets auto-repaint
Button("Toggle", [&](){
    isActive.set(!isActive.get());
});

State-aware properties

WidgetMethodDescription
TextsetText(State<T>)Bind displayed string to any state value
TextsetText(State, true, false)Toggle between two strings on bool state
TextsetTextColor(State, tc, fc)Toggle text color on bool state
ContainersetBackgroundColor(State, tc, fc)Toggle background color on bool state
Tip: State binding causes either a layout pass or a paint-only pass depending on whether the change affects widget size. Text content changes trigger layout; color changes are paint-only.
